## Workflow

1. Create an issue and explain the new issue/feature and assign and add label as appropriate
2. Check out to new branch named "feature/issue-number"
3. Commit as much as you need, with commit message "issue [issue number]: short description of commit"
4. Create a pull request with the issue number in the title
5. Deal with conflicts if any, and request code review
6. Merge request after the review
7. Close issue
8. Reopen issue and repeat the process as neccessary
